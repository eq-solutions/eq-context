## Canonical routing gap ‚Äî found and fixed

**Problem:** `tenant_routing` table was missing from eq-canonical. All canonical
write-through from EQ Service and Quotes was silently swallowed by `TenantNotFoundError`.

**Fix chain:**
1. Created `tenants` + `tenant_routing` tables in eq-canonical (seeded from `organisations`)
2. Generated new 32-byte `TENANT_ROUTING_MASTER_KEY`, set in eq-shell Netlify
3. Built + ran `scripts/provision-sks-tenant.mjs` ‚Äî AES-256-GCM encrypted service-role key,
   inserted sks ‚Üí sks-canonical routing row
4. Deleted provision file after run
5. Triggered eq-shell redeploy via git push

**Result:** canonical-api can now route `X-Tenant: sks` PUT requests to sks-canonical.
`syncCustomer`, `syncSite`, `syncAsset`, `syncDefect`, `emitEvent` from EQ Service,
and `write_event` from EQ Quotes, are live end-to-end.

## eq-shell build fix

Build was failing in `@eq/confirm-ui` DTS compilation. Two related TS errors:

1. `"cap_exceeded"` missing from `ValidationError` union in `eq-validation/validate.ts`
   ‚Äî added `| { kind: 'cap_exceeded'; field: string; reason: string }`
   ‚Üí commit `ecd75c2`

2. Adding `cap_exceeded` made the switch exhaustive; TypeScript then narrowed `e` to
   `never` in the `default:` branch, making `e.kind` fail. Cast to `{ kind: string }`
   in the default branch.
   ‚Üí commit `163d799`

Build now passing. Production deploy live at `core.eq.solutions`.
