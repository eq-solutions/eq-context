# 2026-06-04 — EQ Field: remove the "Pick a demo tenant" workspace picker

**Built by:** Royce Milmlow + Claude Code

## Shipped
- **v3.5.72** (PR [#185](https://github.com/eq-solutions/eq-field/pull/185), squash-merged, live on eq-solves-field.netlify.app — prod `sw.js` = `eq-field-v3.5.72`).
- Removed the v3.5.18 "Pick a demo tenant" overlay. A plain load of the EQ Field host now boots straight into the default `eq` tenant instead of showing the tiered demo chooser.

## What changed
- `index.html` — deleted the `tenant-picker-overlay` HTML block and the boot IIFE that set `display:flex` + `window.__EQ_TENANT_PICKER_ACTIVE__`. The flag was only ever set, never read.
- `styles/base.css` — removed the dead `#tenant-picker-overlay` / `.tpo-*` rules.
- Version bump tuple: `APP_VERSION` (app-state.js) + `sw.js` banner/`CACHE` + in-HTML changelog banner.

## How it works now
- `_detectTenantSlug()` (app-state.js) already defaults the `eq-solves-field` host to the `eq` tenant, so removing the picker means a no-param load resolves to `eq` and proceeds to the PIN gate.
- The Advanced/Enterprise demos remain reachable on demand: `?tenant=demo-trades` (Advanced) and `?tenant=melbourne` (Enterprise) are still allowlisted for the Field host.
- The `#sh=` shell handoff path is untouched.

## Decisions
- **EQ Field is the front door for tenants** — Royce: drop the chooser, build out the Enterprise surface later rather than gating entry behind a demo-tier picker.
- **Soft removal of reach, not capability** — the tier demos still exist, just no longer surfaced as a landing screen.
