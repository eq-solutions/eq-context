## v3.5.99 — Sentry EQ-FIELD-3 + EQ-FIELD-4 (shipped in previous session, resolved today)

**EQ-FIELD-3** — `ReferenceError: isLeave is not defined` (dashboard.js:77, 15 events)
- Root cause: `isLeave()` in lazy-loaded `roster.js` called from core-loaded `dashboard.js`
  before `roster.js` had loaded (landing on dashboard tab, or EQ_LAZY .catch fallback).
- Fix: Added `_isLeave` inline fallback using `LEAVE_TERMS` from eagerly-loaded `app-state.js`.
  Same pattern as the `updateTopStats` guard added in v3.5.84.

**EQ-FIELD-4** — `ReferenceError: auditLog is not defined` (timesheets.js:419, 1 event)
- Root cause: `auditLog()` called unguarded from 14+ modules; `audit.js` was lazy
  (only loaded on Audit tab). Any module calling `auditLog()` before the Audit tab
  was visited crashed.
- Fix: `audit.js` (233 lines) core-loaded in `index.html`; removed from `TAB_SCRIPTS`
  in `lazy-loader.js` to prevent duplicate `let` redeclare crash on second inject.

Both resolved in Sentry with reason notes. Last seen: 8h and 14h before v3.5.99 deployed
— no new occurrences since.

## v3.5.100 — Sentry EQ-FIELD-5 (shipped 2026-06-08)

**EQ-FIELD-5** — `ReferenceError: siteColor is not defined` (dashboard.js:157, 1 event)
- URL: `https://field.sks.eq.solutions/` (EQ Shell embedding Field, embedded=true)
- Root cause: `siteColor()`, `getSiteName()`, and `isAbsence()` all live in lazy-loaded
  `roster.js`, but are called from core-loaded `dashboard.js` before `roster.js` loads.
  Same class of bug as EQ-FIELD-3.
- Fix: Added inline fallbacks for all three functions in `dashboard.js` using only
  eagerly-loaded constants from `app-state.js`:
  - `_siteColor` — uses `SITE_COLOR_MAP`, `EDUCATION_TERMS`, `LEAVE_TERMS`, hash fallback
  - `_getSiteName` — uses `STATE.sites`
  - `_isAbsence` — uses `_isLeave` + `_isEducation`
- Pre-empts the next two crashes in the same render path (getSiteName:158/104/115/116,
  isAbsence:228/231).
- PR #230, merged, live. Smoke confirmed: `dashErr=null` with all 3 functions undefined
  (roster.js not loaded) — fallbacks fired cleanly.

**PR:** [#230](https://github.com/eq-solutions/eq-field/pull/230) — merged, live.
**Production verified:** `sw.js` shows `v3.5.100`.
