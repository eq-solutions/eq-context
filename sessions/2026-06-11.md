# 2026-06-11 — EQ Field: SKS canonical DB wiring (v3.5.125)

**Built by:** Royce Milmlow + Claude Code

## Problem

`core.eq.solutions/sks/field` showing all zeros + permanent "Loading data…" spinners. Root cause: after the v3.5.120 greenfield cut-over to ehow (`ehowgjardagevnrluult`), only 4 of 11 `app_data.field_*` views existed. Field's JWT data path (`DATA_JWT_ENABLED=on`) routes all queries to `app_data.field_*`; missing ones caused PGRST205 on every data load.

## Shipped

- **v3.5.125** — [PR #267](https://github.com/eq-solutions/eq-field/pull/267), squash-merged `59805c1ae4b4e28e44629ef6de564aa2878e5879`, live on `field.eq.solutions` + `field.sks.eq.solutions`.

## What changed

**Migration:** `supabase/migrations/20260611_sks_canonical_field_sync.sql` (idempotent, applied to ehow):

1. **`public.organisations` stub** — Field boot queries `{SB_URL}/organisations?slug=eq.sks`; table was missing → 404 console noise. Created with single SKS row (`id=00000000-...-02`, `slug=sks`).
2. **`public.site_diaries` table** — full schema matching ktmj; completes `JWT_TABLES` coverage.
3. **7 missing `app_data.field_*` views** — `field_schedule`, `field_timesheets`, `field_leave_requests`, `field_audit_log`, `field_prestarts`, `field_toolbox_talks`, `field_site_diaries` (all `SELECT *` pass-throughs, auto-updatable).
4. **RLS WITH CHECK hardening** — all 14 write policies on operational public tables now enforce `org_id = '00000000-...-02' AND auth.role() = 'authenticated' AND app_metadata.tenant_id = '7dee117c-...'`. Previously WITH CHECK was NULL → any authenticated JWT could INSERT rows with any org_id.
5. **`audit_log` policy fix** — old policies used stale nspb UUID (`1eb831f9`). Replaced with correct SKS org_id; SELECT scoped to org_id (audit data is sensitive).

## Adapter view architecture (KEY)

`field_people`, `field_managers`, `field_sites` are **adapters on EQ Service canonical tables** (`app_data.staff` / `app_data.sites`) — read-only (computed columns block PostgREST writes). Real entity data lives there; Field reads, Service owns. The remaining 8 views are simple pass-throughs on `public.*` (writable).
