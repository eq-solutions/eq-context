## Decisions made
- Adopt `AGENTS.md` alongside `CLAUDE.md` as dual entry points so any LLM
  tool lands on the same rules. Symlinks not used (Windows breaks them).
- Retain `CLAUDE.md` name because Claude Code auto-loads it — keeping it
  is zero cost and preserves that integration.
- Context files are written in CommonMark with YAML frontmatter only —
  no tool-specific extensions, no XML tags.

## Lessons learned
- `claude.ai/chat/...` session links expire and leak session identifiers;
  removed from all changelogs. Going forward, reference sessions by ISO
  date in `sessions/YYYY-MM-DD.md` instead.
