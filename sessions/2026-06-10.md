### Key Technical Lesson

`REVOKE ALL ON FUNCTION ... FROM anon` only removes an *explicit* grant to the anon role. Every Postgres function gets an implicit EXECUTE grant to `PUBLIC` at creation; anon inherits from PUBLIC. The advisor fires on inherited grants too. Correct pattern: `REVOKE ALL ON FUNCTION ... FROM PUBLIC` (then `GRANT EXECUTE ... TO authenticated` where needed).
