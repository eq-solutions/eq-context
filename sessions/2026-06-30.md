## Decided
- **Audit classified by DOMAIN, not storage location.** Running-the-business events → Tenant Activity Log (tenant plane, tenant admins). Auth/security plumbing → Platform Security Log (control plane, operator only).
- **The control plane is NOT part of the tenant shell's audit log.** An invite is "running the business" → its activity record goes to the tenant plane even though the identity row is on jvkn.
- **The audit boundary = the canonical-ownership line.** `app_data` (~100 tables) is NOT "canonical"; Shell audits only the spine it owns (customers/sites/contacts/staff/assets). Field/Service/Quotes/GM own their own audit.
- **Mechanism: DB triggers + full before/after snapshot** (Royce chose over app-level writes) — base-table trigger catches every writer exactly once, closing the crm-write/entity-patch dual-path gap.
- **Platform Security Log deferred** to a future operator console.

## Deferred
- Verify header→GUC actor capture on first real UI edit (the one unverified link; graceful degradation if it fails — what-changed still logs).
- Team & access events (invites/role-changes) as app-level writes to the tenant plane.
- Link-table triggers (contact_customer_links, contact_site_links).
- Platform Security Log / operator console.

## Notes
- `supa_audit` extension NOT available on ehow; `pgaudit` available but logs to files not a queryable table — so hand-rolled the supa_audit pattern in a migration (cleaner for One Pipe governance anyway).
- `staff_field_sync` trigger does NOT exist on zaap (verified) — the disable migration is a no-op there.
- `fn_audit()` is best-effort: an outer EXCEPTION handler swallows any audit failure so it can never block a canonical write. Guarded uuid cast so a malformed actor header doesn't lose the row.
