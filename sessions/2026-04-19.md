## Follow-up session — Delta/Equinix work-order Excel import (EQ Solves Service)

Built end-to-end import flow that converts the monthly Delta work-order
`.xlsx` Equinix sends from Maximo into EQ Service maintenance checks.
Shipped on feature branch `feat/delta-wo-import` (commit `e392065`,
pushed to `origin`).

### What was done

- **Migrations** applied to `eq-solves-service-dev` (`urjhmkhbgaxrofurpbgc`):
  - `0049_job_plan_aliases` — tenant-scoped alias table
    (`source_system: delta|maximo|manual`) for remapping unknown
    upstream codes and accepted fuzzy matches. RLS enabled with
    tenant-scoped policies.
  - `0050_check_assets_wo_unique_idx` — partial unique index on
    `check_assets(tenant_id, work_order_number)` WHERE
    `work_order_number IS NOT NULL` as DB backstop for duplicate
    detection.
- **Parser** (`lib/import/delta-wo-parser.ts`) — pure parsing, no DB /
  no React. Strips `AU0x-` site prefix, splits job plans on the last
  dash, maps Delta frequency suffixes to the EQ enum (`A`→annual,
  `Q`/`3`→quarterly, `M`→monthly, `S`/`6`→semi_annual, `W`→weekly,
  `2`/`5`/`10`→n-yr). Unknown suffix fails closed.
- **Fuzzy match helper** (`lib/utils/levenshtein.ts`) — two-row DP,
  case-insensitive. `closestMatch()` returns null beyond `maxDistance`
  (default 2). Covers the known MVSWBD→MVSWDB typo. Fuzzy matches are
  suggestions only — never auto-applied.
- **Server actions** (`app/(app)/maintenance/import/actions.ts`):
  - `previewDeltaImportAction` — resolves sites, job plans (exact +
    alias + fuzzy), assets by `(site_id, maximo_id)`; flags duplicate
    WO numbers.
  - `commitDeltaImportAction` — wrapped in `withIdempotency`.
    Re-parses the workbook server-side; never trusts the client preview
    payload. Refuses wholesale if any blocker exists. Preloads
    `job_plan_items` per distinct `(jobPlanId, frequencyColumn)` pair.
    Per group: inserts `maintenance_checks` → `check_assets` (with
    WO#) → batched `maintenance_check_items`. Audit log carries the
    `mutationId`.
- **UI** — role-guarded `/maintenance/import` page with upload →
  preview → commit wizard; group cards, blocker/warning banners,
  sticky commit bar. Import button added to `/maintenance` toolbar
  (convention: Import left of Create Check).
- **Report** — `PmAssetSection.workOrderNumber` now renders in the
  asset metadata table alongside Job Plan in the PM Asset Report.
- **Tests** — 27 parser tests + 11 levenshtein tests, all passing.
  `tsc --noEmit` clean.
- **Advisors** — Supabase security advisors: 0 new ERROR-level findings.
  Performance advisors: only INFO-level "unused index" on
  `job_plan_aliases` (expected — new table, no traffic yet).

### Decisions

- Server re-parses the workbook on commit rather than trusting a client
  payload — trades duplicate parse cost for a zero-tampering surface.
- Commit refuses wholesale on any blocker (unresolved site, unresolved
  plan, unknown frequency, unmatched asset, duplicate WO) rather than
  partial-commit. Cleaner UX, easier to reason about.
- Fuzzy matches stay user-confirmed. No silent auto-rewriting of
  upstream codes.
- `feat/delta-wo-import` branched off `origin/main` directly — kept
  separate from `feat/ip-hardening` so the two PRs can be reviewed
  and merged independently.
