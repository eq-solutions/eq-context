## Notes

Every `showLoadingOverlay` call (index.html, batch.js, import-export.js) has a paired hide on both success and catch/finally paths — no orphaned spinner state possible in Field's own code. Realtime reconnect (`scripts/realtime.js`) has a proper capped exponential backoff (1s→30s via `_rtBackoffMs`/`_RT_MAX_BACKOFF`), not a runaway fixed-interval loop — the repeating `closed(1006)/reconnect/closed(1000)` cycle seen in the repro console log is cosmetic churn tied to the same tab-visibility-restore event, not the spinner's source. The actual signal: a minified React error #418 (hydration mismatch) fires from Shell's own bundle at the moment the tab regains focus, pointing at Shell's iframe-wrapper component (likely a focus-triggered refetch pattern, e.g. react-query `refetchOnWindowFocus`) crashing before its own loading state clears.
