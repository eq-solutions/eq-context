# Session 2026-07-23 (2) — eq-solves-service: proved --force isn't viable, verified the rest isn't exploitable, fixed the one that was

## Built
- Ran `npm audit fix --force` in a throwaway git worktree (not the shared checkout) purely to see its actual effect rather than reason about it abstractly. Confirmed: downgrades `next` 16.2.10 → 9.3.3 and `exceljs` 4.4.0 → 3.4.0, vulnerability count goes **up** 5 → 95 (12 low, 56 moderate, 26 high, 1 critical) from the ancient transitive tree it pulls in, and the build fails immediately (`next.config.ts` unsupported on Next 9). Worktree + branch removed, nothing kept.
- Before touching anything else, verified whether the 2 real remaining findings (sharp HIGH, uuid moderate) are actually reachable by this app, not just present in `node_modules`. `sharp` is declared only as `next`'s own optional dependency for its built-in `/_next/image` route — grepped the whole app for `next/image`/`<Image>`/`next/og`/direct `sharp` imports: zero real usage, so that code path never runs. `uuid`'s vulnerable advisory (GHSA-w5hq-g745-h8pq) only fires on v3/v5/v6 calls that pass a `buf` argument; `exceljs` (the only thing pulling it in) calls `uuid.v4()` with no arguments, in one place. Neither is exploitable via anything this app does.
- Also checked whether either is fixable another way: confirmed the *newest* available release of both `next` and `exceljs` still pin the same vulnerable versions — genuine upstream blocker, not a version we happened to be behind on.
- Found one real, free fix in the process: `next` bundles its own internal copy of `postcss` (8.4.31, has a real XSS-in-stringify advisory) separate from our own top-level copy, which was already patched (8.5.20). Added a `package.json` `overrides` entry pointing `next`'s copy at the patched version — no `next` version change, no breaking change. `npm run check` (tsc + build) clean. eq-solves-service PR #586, merged (`fa19450`) on Royce's "merge it", Netlify auto-deploy triggered.
- Ran the proper `/brief` session gate before making this last edit (had skipped straight to editing earlier in a previous close's work; corrected this time) — Royce confirmed with "go ahead" before any file was touched.
- Net result: audit findings on eq-solves-service are down to 2 (sharp HIGH, uuid moderate), both confirmed dormant and both confirmed upstream-blocked — nothing further to do until `next` or `exceljs` bump their own dependency.

## Decided
- Royce: proceed with the postcss override fix ("go ahead", after the brief).
- Royce: merge PR #586.

## Deferred
