## Notes
- **Structural lesson, now written into the audit doc:** an "RLS fix closes the gap" claim only holds for reads that actually go through RLS. Any repo reading the same table server-side with a service-role key (or a `SECURITY DEFINER` RPC) needs its own explicit filter — RLS provides no protection there. Worth checking for this pattern specifically whenever a privacy/RLS fix ships in eq-cards and other repos read the same eq-canonical tables.
