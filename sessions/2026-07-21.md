- Root cause #1: both new call sites (`canon-read.js:159`, `verify-pin.js:604`) invoked `captureServerError()` without `await`. The function awaits an HTTP POST to Sentry's ingest endpoint with a 2s timeout; un-awaited, the in-flight fetch is dropped when the Lambda freezes on return. Independently explains part of the 90-day silence.
- Fixed all five: added `await` to all, and corrected the three mistagged sites to `{ tags: { netlify_function: '<name>' } }`, matching the working pattern already at `verify-pin.js:633`.
- Hit a multi-minute tool-safety-classifier outage mid-edit (every Edit call failing "temporarily unavailable"); waited it out and re-verified every file's diff and `node --check` after resuming rather than trusting partial-session memory.
