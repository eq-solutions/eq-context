# 2026-06-03 — EQ Field: Resources "Remove job"

**Built by:** Royce Milmlow + Claude Code

## Shipped
- **v3.5.53** (PR [#160](https://github.com/eq-solutions/eq-field/pull/160), squash-merged, live on eq-solves-field.netlify.app).
- Resources (Resource Allocation) screen: added a **Remove job** action on both Needs Allocation and Confirmed jobs. Previously you could "+ Add Active Job" but never remove one.
- **v3.5.54** (PR [#161](https://github.com/eq-solutions/eq-field/pull/161), merged, live): BUG-009 fix — Remove job button did nothing because `window.confirm` is swallowed in the eq-shell iframe / iOS PWA standalone. Now routes through the shared `#modal-confirm` dialog (`_raConfirm`).

## How it works
- `scripts/sks-pipeline-resource.js → removeProject()` sets `tenders.archived_at` — a reversible soft-remove matching the existing pipeline "kill" pattern. Archived jobs drop out of both Resources and the Pipeline (load filters already use `archived_at=is.null`).
- Roster entries already pushed to `schedule` are deliberately left in place — archiving the tender does not unwind the roster.
- Confirm dialog before archiving. Local state (`_enr`, `_noms`, `_pending`, `_ca`, `_siteCodes`, `_wplan`) cleaned on remove.
- Confirmed jobs with no labour slots now still open a panel (`_emptyConfirmedPanel`) so they remain removable.

## Decisions
- **Archive over hard delete** — reversible, leaves pushed roster intact, matches precedent. (Hard delete was offered and declined.)
- **Both sections** get the control.
