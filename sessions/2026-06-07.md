# 2026-06-07 — shift.started enrichment: backfill, durable emitter, prompt fix

**Built by:** Royce Milmlow + Claude Code

## Problem
The EQ Shell dashboard AI-briefing "On Shift Now" panel (`eq-shell/netlify/functions/ai-briefing.ts`)
was always empty for SKS. It reads `shift.started` canonical_events and the prompt told the model
to "extract name and site from payload" — but the live SKS payload held only job-code -> headcount
(`{"assignments":{"SY5":8,...},"scheduled_count":150}`), no names or human site names.

## Key finding
**No `shift.started` emitter exists anywhere** — ruled out across all repos, DB functions, pg_cron,
and edge functions in both planes. The handful of `shift.started` rows in the sks-canonical data
plane (`ehowgjardagevnrluult`, `app_data.canonical_events`) are **seed/demo data**. The two docs the
task referenced (`docs/ai-briefing-emit-enrichment.md`, `eq-context/cross-repo-contracts-*.md`) do
not exist.

## Data planes (verified)
- **SKS roster** lives in `nspbmirochztcjijmcrx` (SKS LIVE op DB): `public.schedule` (one row per
  person/week; text day cells `mon`..`sun` hold a site code or a leave/admin token like A/L, RDO,
  TAFE, SICK, N/A). `public.sites.abbr` -> `public.sites.name` is the code->human-name map.
  Week key format = `DD.MM.YY` of the week-commencing Monday (e.g. `15.06.26`).
- **canonical_events** live in the separate `ehowgjardagevnrluult` plane (SKS tenant_id
  `7dee117c-98bd-4d39-af8c-2c81d02a1e85`). Any durable writer needs a cross-project hop — done here
  over HTTP via the eq-shell canonical-api, not a direct DB link.

## Shipped — Round 1 (prior session, "backfill now")
- Built `public.eq_field_shift_payload(p_week, p_day, p_on_shift_limit default 6)` in SKS-live.
  Returns the existing shape PLUS `sites` (code->name) and `on_shift` ([{name,site}], spread
  one-per-busiest-site, bounded). SECURITY INVOKER; EXECUTE -> service_role only.
- Backfilled seed rows (ids 22 thu, 26 fri) additively and inserted one fresh in-window event
  (id 31, `_demo_seed: true`) so the panel populates. Cleared `briefing_cache`.
