# 2026-06-05 — Field PostHog pageview fix

**Shipped:** v3.5.77 (PR #190, merged, prod live `eq-field-v3.5.77`).

**What:** Fixed PostHog collapsing ~80% of pageviews under the bare path `/`. Field is a SPA; pageviews were double-captured (autocapture `/` + manual `/#page`). Set `capture_pageview: false` so the manual virtual-URL capture is the single source (one pageview per screen), and added a consecutive-duplicate guard in `pageViewed()` to kill the boot `showPage()`+`mobileNav()` double-fire. Analytics-only, no auth.

**Decisions:** (1) Keep the manual `/#page` virtual-URL scheme (already distinguishes screens in PostHog) rather than drop it — dropping it would collapse everything to one `/` per session. (2) Accept loss of the pre-login gate-bounce `/` pageview as a trade-off; `session_started` + `$pageleave` still cover it. Re-add an explicit `gate` pageview later if bounce-rate matters.

**Gotcha logged:** branched-from-stale-main — the worktree HEAD was behind `origin/main`; reset to clean main before starting. Then #189 took v3.5.76 mid-review → rebased + renumbered to v3.5.77.

**Deferred:** none.
