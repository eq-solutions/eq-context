## GitHub API

**Problem:** Inline `-d` flag fails for large JSON payloads in GitHub API PUT requests.
**Fix:** Write full JSON payload (base64-encoded content) to a temp file, use `--data @/tmp/payload.json`.
**Also:** Always include `branch` parameter explicitly. Include existing file's blob SHA in PUT requests or the API rejects the update.

## SVG Uploads to Claude Project

**Problem:** Claude.ai blocks SVG file uploads for security reasons.
**Fix:** Rename SVG files to .txt before uploading. Rename back after downloading.
**Applies to:** All SVG source files — EQ logo variants and any other vector assets.

## ASIC Connect

**Problem:** ASIC Connect doesn't work in modern browsers.
**Fix:** Use Edge in Internet Explorer compatibility mode.
**Also:** Leave the Building/Property name field blank for standard residential addresses —
filling it in causes validation errors.

## pptxgenjs — PowerPoint Compatibility

**Problem:** Files generated by pptxgenjs open fine in LibreOffice but PowerPoint throws a corrupt file error.
**Root cause:** Empty directory entries in the ZIP structure that LibreOffice ignores but PowerPoint rejects.
**Fix:** Strip empty directory entries from the ZIP after generation — skip any entries ending in `/`.

## docx Library — Page Numbers

**Problem:** `PageNumber` is not a direct named export from the `docx` npm library.
**Fix:** Use `SimpleField("PAGE")` instead.

## ReportLab — Subscripts/Superscripts

**Problem:** Unicode subscript/superscript characters (₀₁₂, ⁰¹²) render as solid black boxes
in ReportLab PDFs. Built-in fonts don't include these glyphs.
**Fix:** Use ReportLab's XML markup tags in Paragraph objects: `<sub>2</sub>` and `<super>2</super>`.
For canvas-drawn text, manually adjust font size and position instead.

## Email Security — ZIP Files with Executables

**Problem:** ZIP files containing .bat, .exe, or Python scripts are flagged by corporate email
security (and SKS specifically). Recipients can't open them.
**Fix:** Remove all executable files from distributions. Replace .bat launchers with plain text
instructions. Better yet, use single-file HTML + Cloudflare Worker pattern — no executable content at all.

---

## ThreatLocker on Corporate PCs

**Problem:** ThreatLocker blocks Python execution, .bat files, and unapproved executables on SKS corporate laptops.
**Fix:** Single-file HTML approach — opens in any browser, zero install, nothing for ThreatLocker to block.
If even the HTML file is blocked: host it on Cloudflare Pages and share a URL instead.

## SheetJS for Client-Side Excel

**Lesson:** SheetJS (xlsx.full.min.js from cdnjs) generates valid .xlsx files entirely in the browser.
No server, no Python, no openpyxl needed for single-file tool distribution.
CDN: `https://cdnjs.cloudflare.com/ajax/libs/xlsx/0.18.5/xlsx.full.min.js`
**Limitation:** Complex cell formatting (borders, fills, merged cells) requires `aoa_to_sheet` + manual
cell style objects. SheetJS community edition has limited style support — use openpyxl server-side
for pixel-perfect templates.

## localStorage Data Design

**Lesson:** Design localStorage data structures as if they're going into Supabase from day one.
Use UUIDs as IDs (`crypto.randomUUID()`), ISO date strings, consistent field names.
The migration from localStorage → Supabase then becomes replacing read/write functions only —
no data reshaping needed.

## Cloudflare Worker CORS

**Lesson:** Always include explicit CORS headers in the worker response AND handle OPTIONS preflight.
Without OPTIONS handling, browsers block the first request before it reaches the worker.
Pattern:
```js
if (request.method === "OPTIONS") {
  return new Response(null, { status: 204, headers: CORS_HEADERS });
}
```

## Supabase MCP — Confirming Project Before Queries

**Lesson:** Always state which Supabase project a query targets before executing it, even in context
where it seems obvious. The cost of confirming is zero. The cost of running a query against the
wrong project (especially SKS live data) is not zero.

## HHT Crypto — CGT Investor Classification

**Lesson:** Hexican Holdings Trust is classified as a CGT investor across ALL crypto holdings —
not trading stock. This applies universally to HHT, personal, and SMSF holdings.
Capital losses in HHT are quarantined within the trust — they cannot flow to personal returns or CDC.
Do not suggest strategies that assume these losses are available elsewhere.
