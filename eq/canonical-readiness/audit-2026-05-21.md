# EQ Shell — Honest audit (2026-05-21)

This is the punch list. Severity-ranked. The agent (Claude) walked every
URL on `core.eq.solutions` while signed in as `dev@eq.solutions` (manager
+ platform_admin) with the seed data live on the `core` tenant.

Triage convention:
- 🔴 **Broken** — something explicitly doesn't work / blocks usage
- 🟡 **Clunky** — works, but visibly unfinished / off-brand / not striking
- 🔵 **Polish** — works fine, would be sharper with a small change

## Detailed walk — page by page

### `/` (login when signed out)

Could not verify directly during this audit pass — agent is signed in,
so `/` redirects to `/core`. **Past screenshots confirm the dark-navy
hero treatment with the "BUILT IN SYDNEY" eyebrow + Sky-accent
headline + canonical eQ logo lands correctly.** That's the visual
reference point everything else should match.

### `/core` (tenant home)  — 🟡 BIGGEST GAP

**Works functionally.** Shows real data. Modules grid populated. Recent
intake activity row is populated with last 2 intakes from yesterday.

**Visually disconnected from login.** Login is dark-navy, big confident,
Sky accents — looks like the operating system. Home is white-on-white
admin dashboard with pale gray stat cards — looks like Stripe billing.
A user moving from login → home should feel "you're now inside the
platform". They feel "you're now inside a generic dashboard."

Specific issues:
- Headline "Welcome back, Dev" is friendly. Could be louder — e.g.
  "Operating EQ Solutions" or "Your platform" or showing real-time
  state ("3 pending approvals · 2 intakes this week")
- Stat cards are uniform 200px boxes with gray text — should have
  visual hierarchy (the most important number bigger; secondary
  metrics smaller)
- No use of brand Sky anywhere on the page (except in the role pill
  at top right). Login uses Sky as the accent that ties everything
  together; home doesn't.
- "Welcome back, Dev" → "Dev" is parsed from `dev@` email prefix.
  Reads weirdly. Could pull the user's actual `first_name` from
  staff record if linked.
- Below the modules grid there's ~700px of white space. Page feels
  short / unfinished.
- Recent activity rows: dots are gray (status='completed' string
  mismatch — see finding #4).
- "+50 this week" on Customers makes sense (data was seeded this
  week), but execs won't realise that. Could read as "the platform
  added 50 customers this week" which sets weird expectations.

### `/core/intake`  — 🔴/🟡

Loads after **~8 seconds**. No topbar (cannot navigate away without URL
bar). Shows the old SimPRO surface: Quick export (SimPRO → Outlook etc),
Bundle → destination paste, Save into EQ.

Three issues stacked:
- 🔴 **No topbar** — once you're here, the only way out is the back
  button or typing a URL
- 🟡 **SimPRO-specific copy everywhere** ("Drop a SimPRO file, pick
  where it's going") — feels like a beta tool, not the canonical
  intake module
- 🟡 **No nav to the new per-domain pages** at
  `/core/intake/{core,field,quotes,cards,service}` — users won't
  discover the 42-entity intake surface

### `/core/intake/core`  — 🔴 BROKEN

**Page never finishes loading.** Document_idle timeout exceeds 45
seconds. The browser tab becomes unresponsive — subsequent navigations
to other pages on the same tab also hang.

Root-cause hypothesis: `EntityImportPanel.tsx` does **20 static JSON
imports** at module-load time + imports `@eq/confirm-ui` which pulls
in `pdfjs` (~1.6MB compressed, ~5MB uncompressed). When the user
lands on `/core/intake/core`, all 20 schemas + pdfjs parse into the
main thread, blocking. Bundle output confirms this: there's a
`site.schema-XXX.js` chunk at 405 KB which is one of the JSON
schemas — Vite is bundling all the JSON imports synchronously.

Fix path:
1. Switch the JSON imports to `import()` lazy-loads triggered on the
   "Import CSV" click (not module load)
2. Code-split `EntityImportPanel.tsx` into a separate lazy chunk
3. Lazy-load `@eq/confirm-ui` on-demand only when a user clicks
   import on an entity card

**Likely affected:** all 5 per-domain landing pages
(`/core/intake/{core,field,quotes,cards,service}`), since they all
import the same EntityImportPanel.

## Cross-cutting findings

### Bundle weight

The main bundle is **694 KB compressed**. Main culprit is `@eq/intake-demo`
+ `pdfjs` + 20+ JSON schemas statically bundled. The site.schema chunk
alone is **405 KB**. This is the proximate cause of finding #1
(EntityImportPanel hang) and contributes to the slow `/core/intake`
load (#3).

Fix path: aggressive code-splitting + lazy-loading per entity card,
not per page.

### Brand consistency drift

- Login: dark-navy, Sky-accent, eq.solutions energy ✅
- Topbar: clean Sky+ice navigation ✅
- Home: white admin dashboard ❌
- Intake legacy: separate aesthetic (SimPRO copy) ❌
- Admin audit: Sky table header per spec ✅
- Entity browser: Sky table header per spec ✅
- Domain landing pages: not verified, but they use a Topbar so likely ok

The home + intake-legacy disconnects are the real consistency gaps.
Everything else aligns.

### "Canonical" word scrubbed from UI ✅

The 3 user-visible occurrences are fixed. Comments + variable names
still use "canonical" but those aren't visible.

### Module entitlements ✅

After backfilling for `core`, all 6 modules show in the home grid
(Intake/Cards/Field LIVE, Quotes/Service/Tender Pipeline SOON).

## Recommended fix order

Picking by impact-on-visible-bar-for-execs:

1. **Fix the `EntityImportPanel` hang** (`/core/intake/{module}` blocking) — 🔴 blocking
2. **Redesign the home screen** to match login energy — 🟡 biggest visible gap
3. **Add topbar to `/core/intake` legacy surface + nav to per-domain pages** — 🟡 user trapped
4. **Fix the "completed" green-dot string mismatch** — 🔵 trivial
5. **Fix the Pending Leave delta inconsistency** — 🔵 small
6. **Token mints tab populating** — 🔵 investigation
7. **Verify the 9 unverified pages** post-fix — 🔵 verification pass

Items #1 and #2 are the highest-leverage. #1 makes the platform usable
end-to-end. #2 makes the platform feel like a real product.
