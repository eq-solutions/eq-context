# eq-solves-service changelog

## 2026-07-23
- **PR #581 — phone-only Shell accounts treated as first-class, not a crash.** A technician with no name/email (mobile phone-only sign-up) crashed the Maintenance page sort (`null.localeCompare`). Fixed with a shared `displayNameFrom(name, email, phone)` helper across Maintenance, Admin Users, Defects, and Audit log — falls back to phone number instead of blank/crashing. `CanonicalMember`/`Asset` types widened to match the real nullable shape.
- **Admin Users duplicate-account cleanup.** Brian Griffin-Colls had two active Shell accounts for the same tenant (phone-only + email-based). Verified zero real references to the stale one across both data planes, then merged. Swept all tenants for the same pattern — no other instances found.
- **PR #582, #584 — Asset # display fixed everywhere it was wrong.** Maintenance check detail, Assets list, and Asset detail pages were all showing the always-empty `maximo_id` field instead of the real, populated `external_id` (Asset # from the customer's import sheet). Fixed in all three places; `external_id` added to the shared `Asset` type. Added the missing Maintenance Plan column and an Export-to-Excel button on the check detail page.
- **PR #585 — scope-funding visibility on the check detail page and in reports.** New `lib/scope-context/asset-funding.ts` batch-resolves each asset's funding status (contracted / ad-hoc / not-funded / excluded) against `contract_scopes`, matching by asset pin first then by job-plan code. Amber "Not funded" badge on the check detail list + a visible count banner; a `hideOutOfScope` toggle (visible-by-default); threaded through to both the Field Run-Sheet/Print-Blank report and the Customer report, each stamping an amber "(NOT FUNDED)" marker inline rather than silently dropping the asset unless the toggle says otherwise.

- **PR #591 MERGED — ACB/NSX Create Check screens were saving frequency as plain English ("Annual", "5 Yearly") instead of the short code ("annual", "5yr") every other part of the app expects, with no validation at all on that save.** One live check had already gone in wrong (a DigiCo NSX check) — confirmed against the live database before fixing, then corrected as part of the same migration (0192).
